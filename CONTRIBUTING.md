# Contributing to Claude Code RFCs

Thank you for your interest in contributing to Claude Code RFCs! This document provides guidelines for contributing.

## Ways to Contribute

### 1. Discuss Existing RFCs

- **Comment on GitHub Issues**: Share your thoughts on existing proposals
- **Provide Use Cases**: Real-world scenarios help strengthen proposals
- **Report Problems**: Point out potential issues or edge cases

### 2. Propose New RFCs

Have an idea for improving Claude Code? Here's how to propose it:

1. **Check Existing RFCs**: Make sure your idea isn't already proposed
2. **Open a Discussion**: Start with a GitHub Discussion to gauge interest
3. **Write the RFC**: Use our [template](./template.md)
4. **Submit a PR**: Create a pull request with your RFC

### 3. Improve Existing RFCs

- Fix typos or clarify language
- Add missing details or examples
- Translate to other languages
- Create diagrams or visualizations

## RFC Format

Each RFC should include:

```markdown
# RFC-XXXX: Title

- **Status**: Draft | In Review | Accepted | Implemented | Rejected
- **Author**: Your Name
- **Created**: YYYY-MM-DD

## Summary
Brief overview (2-3 sentences)

## Motivation
Why is this needed? What problem does it solve?

## Detailed Design
Technical details of the proposal

## Benefits
What advantages does this provide?

## Alternatives Considered
Other approaches and why they were rejected

## Challenges
Known difficulties and possible solutions
```

## Translation Guidelines

We welcome translations! When translating:

1. Create a new file: `README.<lang>.md` (e.g., `README.ja.md`, `README.zh-tw.md`)
2. Keep the same structure as the original
3. Translate content, not code examples
4. Add yourself as translator in the header

Supported languages:
- English (primary)
- Traditional Chinese (zh-tw)
- Japanese (ja)

## Code of Conduct

- Be respectful and constructive
- Focus on ideas, not individuals
- Welcome newcomers
- Assume good intentions
